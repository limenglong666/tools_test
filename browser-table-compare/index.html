3 · beaconcha.in Validator History（CL）

对比区间：457380 – 457394 · 共 15 个 Epoch · 时间 —

示例验证器 index：1283884 · beaconcha.in 页面

在 beaconcha.in 验证器页面打开 Console 运行 beaconcha-extract.js， 或从 Network 面板复制 incomedetailhistory 接口 JSON。

从 beaconcha.in 页面 Validator History 表格直接复制粘贴，每行一条。 格式示例：457448 +8609 GWei（Tab 分隔，含 GWei 单位）。

粘贴后自动预览解析条数与 CL 合计。

支持的粘贴格式

beaconcha 页面复制（推荐）
457448 +8609 GWei
457447 +8612 GWei
或 ETH 小数
457394 0.000012345
以 # 开头的行会被忽略。

4 · Etherscan EL（Transfer）

在 Etherscan 地址页 的 Transactions 中，筛选 Method = Transfer 且时间在上方 Epoch 对应区间内（由主网 genesis 公式换算）的转入，求和后除以 CSV 该 Epoch 区间内唯一 Public Key 数量，再与所选行 CSV EL 对比。

Epoch 时间范围将随上方起始/结束 Epoch 自动更新。

Epoch → 时间说明

主网：timestamp = 1606824023 + epoch × 32 × 12（秒，UTC）。区间结束时间为下一 Epoch 起始前 1 秒。 也可在 beaconcha.in/epoch/{n} 查看各 Epoch 时间作对照。

EL 计算说明

Etherscan 侧：在监控地址的 Transactions 中，取 Method 为 Transfer（含原生 ETH 转入）且 timeStamp 落在 Epoch 区间对应 UTC 时间内的交易，对 value 求和，再除以 CSV 中 Epoch Range 与该区间有交集的唯一 Public Key 数量，得到人均 EL。 CSV 侧：所选行的 EL Rewards Generated（单验证器）。
